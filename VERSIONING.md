# winC2D 版本管理指南

## 分支结构

### main（主开发分支）
- **用途**: Tauri + Rust 新版本的主要开发分支
- **版本号**: v2.x.x
- **发布政策**: 定期发布新功能和改进
- **保护**: 启用 PR review
- **CI/CD**: Tauri 构建工作流

### csharp-legacy（维护分支）
- **用途**: C# + WinForms 旧版本的维护分支
- **版本号**: v1.x.x
- **发布政策**: 仅接收安全补丁和紧急 bugfix
- **保护**: 严格，仅允许维护者合并
- **CI/CD**: dotnet 构建工作流（可选）
- **状态**: 不再添加新功能

## 版本号规范

### C# 版本 (v1.x.x)
- v1.5.0 - 最后的功能版本
- v1.5.1+ - 仅限安全补丁
- 示例: v1.5.2（安全补丁）

### Rust 版本 (v2.x.x)
- v2.0.0-beta - 测试版本（2026年6月）
- v2.0.0 - 正式版本（2026年9月）
- v2.1.0, v2.2.0... - 后续更新

## Release 标记规范

```
v1.5.0 - C# 最后一个版本
v2.0.0-beta.1 - Rust 测试版本
v2.0.0 - Rust 正式版本
```

## 用户迁移指南

### 时间表

| 时间 | 操作 |
|------|------|
| 现在 | C# v1.5.0 发布，进入维护模式 |
| 2026年6月 | Rust v2.0-beta 发布，邀请测试 |
| 2026年9月 | Rust v2.0.0 正式发布 |
| 2027年后 | C# v1.x 仅接收安全补丁 |

### 现有用户的选择

1. **继续使用 C# 版本**
   - 在 csharp-legacy 分支获取最新代码
   - 下载 v1.5.x 版本
   - 将接收 12+ 个月的维护支持

2. **升级到 Rust 版本**
   - 等待 v2.0.0 正式版（2026年9月）
   - 获得性能提升和跨平台支持
   - 在 v2.0-beta 阶段可以抢先体验和反馈

## GitHub 配置

### 分支保护规则

#### csharp-legacy
- ✅ 需要 PR review（1 个审批者）
- ✅ 需要通过状态检查
- ✅ 限制推送权限（仅维护者）
- ❌ 禁止强制推送

#### main
- ✅ 需要 PR review（1 个审批者）
- ✅ 需要通过 CI/CD
- ❌ 允许管理员跳过规则（用于紧急发布）

### Release 发布流程

1. 更新版本号
2. 更新 CHANGELOG
3. 创建 Git Tag（格式: v2.0.0）
4. GitHub Actions 自动构建
5. 手动创建 Release 并上传制品

## FAQ

### Q: 我现在应该下载哪个版本？
**A**: 
- 稳定性优先 → C# v1.5.x
- 新功能/性能 → Rust v2.0-beta（测试）或等待 v2.0.0

### Q: C# 版本还会更新吗？
**A**: 仅接收安全补丁和紧急 bugfix，不添加新功能

### Q: 两个版本能同时运行吗？
**A**: 可以，两个版本是独立的应用程序

### Q: 如何从 C# 版本升级到 Rust 版本？
**A**: 等待升级指南（将在 v2.0 release notes 中提供）
